# hello-aakash-site

Static site generated by `generate.py`, containerized with nginx, auto-deployed to **Google Cloud Run** via GitHub Actions on every push to `main`. No Firebase — pure GCP.

## Files

- `generate.py` — builds `site/index.html` and `site/style.css`
- `Dockerfile`, `nginx.conf` — packages the generated site into an nginx container listening on port 8080 (required by Cloud Run)
- `.gcloudignore` — controls what gets uploaded to Cloud Build (keeps the generated `site/` folder even though it's git-ignored)
- `.github/workflows/deploy.yml` — CI: runs `generate.py`, then builds and deploys the container to Cloud Run

## One-time setup

1. **Enable required APIs** on your GCP project (Console → APIs & Services → Enable APIs): Cloud Run API, Cloud Build API, Artifact Registry API.

2. **Create a service account** for GitHub Actions: GCP Console → IAM & Admin → Service Accounts → Create service account. Grant it these roles: Cloud Run Admin, Cloud Build Editor, Artifact Registry Writer, Service Account User, Storage Admin.

3. **Create a JSON key** for that service account (Keys tab → Add Key → JSON) — it downloads automatically.

4. **Add it as a GitHub secret**: repo → Settings → Secrets and variables → Actions → New repository secret → name `GCP_SA_KEY` → paste the entire JSON file contents.

5. **Replace `YOUR_GCP_PROJECT_ID`** in `.github/workflows/deploy.yml` with your real GCP Project ID. Adjust `REGION`/`SERVICE` there if needed.

6. **Push to GitHub:**
   ```
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/<your-username>/<your-repo>.git
   git push -u origin main
   ```

Every push to `main` runs `generate.py`, builds the container, and deploys it to Cloud Run. The Actions log (and the Cloud Run console) will show the live service URL, e.g. `https://hello-aakash-site-xxxxx-uc.a.run.app`.
